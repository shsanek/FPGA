# Hardware Blitter

Аппаратный ускоритель отрисовки текстур для DOOM, встроенный в SCRATCHPAD.

## Архитектура

Блиттер — FSM внутри `SCRATCHPAD.sv`, имеет доступ к шине как bus master.
Все операции (чтение текстур, чтение colormap, запись пикселей) идут через `PERIPHERAL_BUS`.

```
CPU записывает MMIO регистры → BLIT_CMD запускает FSM →
  для каждого пикселя:
    1. FETCH_TEX:  чтение текселя из DDR (через кэш)
    2. FETCH_CMAP: чтение colormap из SCRATCHPAD (через шину)
    3. WRITE_PIXEL: запись пикселя в SCRATCHPAD (через шину)
  → CPU stall снимается после S_DONE
```

При `blitter_active=1` блиттер захватывает шину, CPU стоит на pause.

## MMIO регистры (base = SCRATCH_BASE + 0x20000)

| Offset | Имя | R/W | Описание |
|--------|-----|-----|----------|
| 0x00 | CMD | W | Команда: 1=column, 2=span, 0=idle |
| 0x04 | STATUS | R | bit0: busy |
| 0x08 | SRC_ADDR | W | Адрес текстуры (30-bit bus addr) |
| 0x0C | SRC_FRAC | W | Начальная дробная координата (16.16 fixed) |
| 0x10 | SRC_STEP | W | Шаг по текстуре (16.16 fixed) |
| 0x14 | SRC_MASK | W | Маска текстурного индекса (127 для column, 63 для span) |
| 0x18 | DST_OFFSET | W | Смещение в screen buffer (байты от начала SCRATCHPAD) |
| 0x1C | DST_STEP | W | Шаг назначения (SCREENWIDTH для column, 1 для span) |
| 0x20 | COUNT | W | Количество пикселей |
| 0x24 | CMAP_OFFSET | W | Адрес colormap (полный 30-bit bus addr) |
| 0x28 | SRC_YFRAC | W | Y-координата текстуры для span (16.16 fixed) |
| 0x2C | SRC_YSTEP | W | Y-шаг для span |
| 0x30 | SRC_SHIFT | W | log2(ширина текстуры) для span (обычно 6) |

## Команды

### CMD=1: Column (стены)
Вертикальная колонка. Текстурный индекс: `(frac >> 16) & mask`.
Шаг назначения = SCREENWIDTH (320), пиксели идут вниз по экрану.

### CMD=2: Span (полы/потолки)
Горизонтальная полоска. Текстурный индекс: `(yfrac >>> (16-shift)) & (mask << shift) + (xfrac >>> 16) & mask`.
Используется арифметический сдвиг (`$signed() >>>`) для корректной обработки отрицательных координат.

## Stream cache (bit29)

Софт может установить bit29 в SRC_ADDR (`| BUS_STREAM`) для обхода основного 4-pool кэша
и использования 1-entry stream cache. Тестирование показало что это **не даёт выигрыша** —
4-pool кэш эффективнее за счёт 4 линий vs 1.
