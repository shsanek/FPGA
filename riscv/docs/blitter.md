# Hardware Blitter

Аппаратный ускоритель отрисовки текстур для DOOM, встроенный в SCRATCHPAD.

## Архитектура

Блиттер — FSM внутри `SCRATCHPAD.sv`, имеет доступ к шине как bus master.
Все операции (чтение текстур, чтение colormap, запись пикселей) идут через `PERIPHERAL_BUS`.

```
CPU записывает MMIO регистры → BLIT_CMD запускает FSM →
  для каждого пикселя:
    1. FETCH_TEX:  чтение текселя из DDR (через кэш)
    2. FETCH_CMAP: чтение colormap из SCRATCHPAD (через шину)
    3. WRITE_PIXEL: запись пикселя в SCRATCHPAD (через шину)
  → CPU stall снимается после S_DONE
```

При `blitter_active=1` блиттер захватывает шину, CPU стоит на pause.

## MMIO регистры (base = SCRATCH_BASE + 0x20000)

| Offset | Имя | R/W | Описание |
|--------|-----|-----|----------|
| 0x00 | CMD | W | Команда: 1=column, 2=span, 0=idle |
| 0x04 | STATUS | R | bit0: busy |
| 0x08 | SRC_ADDR | W | Адрес текстуры (30-bit bus addr) |
| 0x0C | SRC_FRAC | W | Начальная дробная координата (16.16 fixed) |
| 0x10 | SRC_STEP | W | Шаг по текстуре (16.16 fixed) |
| 0x14 | SRC_MASK | W | Маска текстурного индекса (127 для column, 63 для span) |
| 0x18 | DST_OFFSET | W | Смещение в screen buffer (байты от начала SCRATCHPAD) |
| 0x1C | DST_STEP | W | Шаг назначения (SCREENWIDTH для column, 1 для span) |
| 0x20 | COUNT | W | Количество пикселей |
| 0x24 | CMAP_OFFSET | W | Адрес colormap (полный 30-bit bus addr) |
| 0x28 | SRC_YFRAC | W | Y-координата текстуры для span (16.16 fixed) |
| 0x2C | SRC_YSTEP | W | Y-шаг для span |
| 0x30 | SRC_SHIFT | W | log2(ширина текстуры) для span (обычно 6) |

## Команды

### CMD=1: Column (стены)
Вертикальная колонка. Текстурный индекс: `(frac >> 16) & mask`.
Шаг назначения = SCREENWIDTH (320), пиксели идут вниз по экрану.

### CMD=2: Span (полы/потолки)
Горизонтальная полоска. Текстурный индекс: `(yfrac >>> (16-shift)) & (mask << shift) + (xfrac >>> 16) & mask`.
Используется арифметический сдвиг (`$signed() >>>`) для корректной обработки отрицательных координат.

## Stream cache (bit29)

Софт может установить bit29 в SRC_ADDR (`| BUS_STREAM`) для обхода основного 4-pool кэша
и использования 1-entry stream cache. Тестирование показало что это **не даёт выигрыша** —
4-pool кэш эффективнее за счёт 4 линий vs 1.

## Бенчмарк (стартовая сцена E1M1, стоим на месте)

| Метрика | Описание | Software | % loop | Blitter | % loop |
|---------|----------|----------|--------|---------|--------|
| bsp | Обход BSP-дерева + отрисовка стен | 80,930 us | 35.8% | 66,130 us | 48.9% |
| planes | Полы и потолки (visplane + span) | 99,400 us | 44.0% | 25,220 us | 18.7% |
| masked | Спрайты и прозрачные текстуры | 7,800 us | 3.5% | 6,020 us | 4.5% |
| setup | Настройка камеры, очистка буферов | 206 us | 0.1% | 242 us | 0.2% |
| pre | Статус-бар, меню, wipe-эффекты | 3,760 us | 1.7% | 3,711 us | 2.7% |
| post | HUD-текст, рамка, отрисовка меню | 39 us | 0.0% | 40 us | 0.0% |
| oled | Downscale 320x200 -> 96x64 + SPI flush | 7,305 us | 3.2% | 7,332 us | 5.4% |
| tics | Игровая логика: AI, физика, движение | 11,540 us | 5.1% | 11,600 us | 8.6% |
| input | Чтение клавиатуры/UART | 1 us | 0.0% | 0 us | 0.0% |

Блиттер даёт **+68% FPS** (4.4 -> 7.4). Основной выигрыш — planes (4x ускорение).

## Планы оптимизации

### Текущие узкие места (Blitter, E1M1)

```
LOOP 135,200 us (7.4 FPS)
├── bsp      66,130 us  48.9%  ← BSP traversal ~50ms (CPU) + стены ~15ms (blitter)
├── planes   25,220 us  18.7%  ← visplane setup ~10ms (CPU) + spans ~15ms (blitter)
├── masked    6,020 us   4.5%  ← спрайты (CPU, без блиттера)
├── tics     11,600 us   8.6%  ← AI, физика, движение (CPU)
├── oled      7,332 us   5.4%  ← downscale 320×200→96×64 (CPU)
├── pre       3,711 us   2.7%  ← статус-бар, меню (CPU)
└── прочее      ~1ms     0.7%
```

### Возможные оптимизации

| # | Оптимизация | Тип | Цель | Ожидаемый эффект | Сложность |
|---|-------------|-----|------|------------------|-----------|
| 1 | **Dual-port BRAM для записи пикселей** | HW | bsp, planes | -2 такта/пиксель (запись 1 такт вместо 3 через шину) | Средняя |
| 2 | **Batch write для span** | HW | planes | Накапливать 4 байта → 1 write вместо 4. -75% записей для span | Средняя |
| 3 | **Masked спрайты через блиттер** | HW+SW | masked | R_DrawMaskedColumn на блиттер. 6ms → ~2ms | Средняя |
| 4 | **OLED scale аппаратно** | HW | oled | DMA из scratchpad в OLED FB. 7ms → ~0ms CPU | Высокая |
| 5 | **Dual-port BRAM для colormap** | HW | bsp, planes | Чтение colormap за 1 такт вместо 3 | Низкая |
| 6 | **BSP/CPU оптимизация** | SW | bsp | Упрощение R_ClipSolidWallRange, lookup tables | Низкая-Средняя |

### Приоритет

1. **(1)+(5) Dual-port BRAM** — самый чистый выигрыш, -4 такта/пиксель (cmap 1 vs 3, write 1 vs 3). ~30% ускорение отрисовки.
2. **(2) Batch write** — дополнительно для span (planes), совместимо с dual-port.
3. **(3) Masked на блиттер** — простое расширение, экономит ~4ms.
4. **(4) OLED аппаратно** — экономит 7ms, но сложная реализация.
5. **(6) CPU оптимизации** — BSP traversal 50ms, но это фундаментальный алгоритм, мало что можно сделать без pipeline CPU.
